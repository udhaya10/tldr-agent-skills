## Empirical Probes
* **Observation:** Tool evaluated and integrated successfully via batch script profiling.

## Architectural Deep Dive
* **Under the hood:** `coverage` is a parser, not a test runner. It reads existing Cobertura XML or LCOV files generated by other tools and maps the coverage percentages onto the `tldr` AST index.
* **Performance:** Instant file parsing.
* **LLM Cognitive Load:** Allows the LLM to see which AST nodes are untested.

## Intent & Routing
* **User/Agent Goal:** Parse existing coverage report files.
* **When to choose this over similar tools:** Use when you need to view coverage metrics, but remember you must run tests externally first.
